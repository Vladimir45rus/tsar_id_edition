# Сборка и запуск «Царь-ID» (мобильное приложение)

## 1. Окружение

- Установите [Flutter SDK](https://docs.flutter.dev/get-started/install) (стабильный канал, Dart 3.3+).
- В корне проекта должен лежать файл `.env` (можно скопировать из `.env.example` и заполнить). Пустой `.env` с комментарием уже включён в репозиторий как asset — при необходимости замените значения своими.

Переменные читаются через `flutter_dotenv` в `lib/main.dart` (ошибка отсутствия файла не падает, если asset на месте).

## 2. Зависимости

```bash
cd "путь/к/tsar_id edition"
flutter pub get
```

Для **Android**: при первой сборке Gradle подтянет `sqlcipher_flutter_libs` и остальные AAR.

Для **iOS**: после `pub get` выполните в каталоге `ios/`:

```bash
cd ios && pod install && cd ..
```

## 3. Запуск в режиме разработки

```bash
flutter devices
flutter run
```

Укажите устройство явно, например:

```bash
flutter run -d <device_id>
```

## 4. Сборка релиза

**APK (универсальный):**

```bash
flutter build apk --release
```

Артефакт: `build/app/outputs/flutter-apk/app-release.apk`.

**App Bundle (Google Play):**

```bash
flutter build appbundle --release
```

**IPA (iOS, требуется Mac + подпись):**

```bash
flutter build ipa --release
```

Перед этим настройте команду разработчика, профили и `ios/Runner.xcworkspace` в Xcode.

## 5. Тестирование офлайн-режима

1. Запустите приложение на устройстве или эмуляторе.
2. Включите режим «в самолёте» или отключите Wi‑Fi и мобильные данные.
3. Убедитесь, что на главном экране отображается жёлтый баннер: «Оффлайн — синхронизация и гео-проверка отложены».
4. Проверьте вкладки **Сейф**, **Пароли**, **Генератор** — они должны открываться и работать с локальными данными без сети.
5. Включите сеть снова: баннер должен исчезнуть после обновления статуса `connectivity_plus`.

## 6. Сброс генератора паролей

- При **переключении нижней вкладки** (с Генератора на другую) вызывается `GeneratorController.clearPasswords()` — сгенерированные пароли очищаются.
- Дополнительно можно нажать кнопку смены длины/опций и снова сгенерировать партию — старый список заменяется новым.

## 7. Автозаполнение и облако

- Полноценное объявление приложения как **системного источника паролей** требует нативных модулей (Android `AutofillService`, iOS Credential Provider). В проекте есть Dart-заглушка `lib/core/autofill/autofill_integration.dart` и комментарии по шагам.
- Прямая загрузка в iCloud Drive / Google Drive из кода без системного диалога ограничена политиками ОС; экспорт бэкапа выполняется через **«Поделиться»** (`share_plus`), пользователь сам выбирает облачное приложение.

## 8. Примечания по безопасности экрана

- На вкладках основной оболочки включён `screen_protector` (скриншоты/запись — по возможностям ОС). На экране **Настройки** защита временно снимается и снова включается после возврата.
- Для экранов **ввода PIN** при необходимости добавьте тот же вызов в соответствующие виджеты (см. `MainShellScreen._setScreenshotProtection`).
